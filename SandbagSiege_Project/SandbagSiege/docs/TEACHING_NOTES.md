# Teaching Notes

## A simple lesson flow

1. Open `index.html` and explain that HTML lays out the page.
2. Open `style.css` and show how colours, spacing, and panels are styled.
3. Open `game.js` and find the `state` object.
4. Explain that the `state` object is the game's memory.
5. Find `update()` and `render()` and explain that the game repeats those jobs every frame.

## Good beginner questions

- What happens if we change the player's health?
- What happens if we make the sniper rifle faster?
- What happens if we add another enemy type?
- What happens if we change the score values?
- What happens if we draw a different sprite frame?

## Easy starter edits for children

- Change the game title text.
- Change weapon names.
- Change enemy colours in the sprite generator.
- Increase or decrease wave length.
- Make grenades stronger.
- Add a new mission message.

## Important programming ideas

- Variables store changing information.
- Objects group related information together.
- Arrays let us keep many enemies and bullets in one list.
- Functions break big jobs into smaller jobs.
- Conditionals help the game make decisions.
- Loops let the game update many objects quickly.
